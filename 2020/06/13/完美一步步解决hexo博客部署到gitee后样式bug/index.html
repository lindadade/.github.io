完美一步步解决hexo博客部署到gitee后样式bug

2020-06-13

✨将hexo博客部署成功到gitee上后
也可以根据gitee Pages服务成功线上访问博客
可是问题一个接着一个来了…

问题来源

✨将hexo博客部署成功到gitee上后
也可以根据gitee Pages服务成功线上访问博客

可是！！本地的样式css和js文件在线上并没有加载出来!

  • 本地跑的hexo博客样式

  • gitee跑的hexo博客样式

查找问题

🌛这个问题是怎么导致的呢？？

  • 第一步：直觉打开开发者工具

    (Google) F12 或 检查

    可以清楚得看到console中报了很多得错误——大致都为css和js文件没有加载出来

  • 第二步：为啥没有加载出来

    粘贴网站进行查看

    完美地404了！！

  • 第三步：猜测

    这些文件应该在哪？

    查看网址的前缀仅仅只是自己gitee的主网

    https://lin_daren.gitee.io/

可是文件应该都在这个博客项目中才对…

将前缀添加自己项目地址变为

https://lin_daren.gitee.io/lindada-blog/{css或js文件}

这样就成功访问到目标文件了！！

这样我们就成功找到样式仅在本地显示而不在gitee线上显示的原因了

解决问题

上度娘去看如何hexo修改线上访问地址的前缀

简单来说只用两步

  • 1.修改_config.yml文件

    将 url: /

    修改为 rul: 图片上框的地址

    将 root: /

    修改为 root: /博客项目名称/

  • 2.重新打包上传hexo

    hexo clean
    hexo g
    hexo d

再遇问题

可是即便这样gitee上Pages也仍没有变化这是为什么呢

找啊找…找啊找…没找到一堆好朋友…掉几根头发…

原来是因为gitee的Pages服务是不会自动更新的需要手动更新

github 就没有这个烦恼 会自动进行更新

解决

点击更新！完美解决🌟🌟
